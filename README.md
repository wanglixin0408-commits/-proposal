Proposal Report
一、動機與目標
在日常生活中，學生與上班族常常需要同時處理多項任務，但現有的待辦清單工具多半缺乏「優先順序管理」與「效率最佳化」功能。因此，本專題希望設計一個結合資料結構的任務管理系統，能夠自動依據任務的重要性與截止時間進行排序，提升使用者的時間管理效率。
本專題的主要目標為：
建立一個有效率的任務管理系統
利用資料結構（如 Heap、Queue、Linked List）提升排序與查詢效率
提供直觀的操作介面（CLI 或簡單 GUI）
二、預期功能
本系統預計具備以下功能：
任務新增與刪除
使用者可以新增任務（名稱、截止時間、優先級）
可刪除已完成或不需要的任務
任務排序（核心）
使用 Priority Queue（Heap） 自動排序任務
優先顯示「最重要或最急迫」的任務
任務查詢
可查詢所有任務
可依條件篩選（例如：今天要做的事）
任務完成標記
標記任務為完成並移除或存入歷史紀錄
資料儲存（進階）
將任務存成檔案（JSON / TXT）
下次開啟可讀取
三、使用技術
本專題將使用以下技術：
程式語言：Python / C++（依課程要求）
資料結構：
Heap（Priority Queue）→ 任務排序
Linked List → 任務儲存
Queue / Stack → 操作紀錄（進階）
其他技術：
檔案處理（File I/O）
簡單 CLI 或 GUI（例如 Tkinter）
四、時程規劃
週數	工作內容
第1週	題目發想與Proposal撰寫
第2週	系統設計（資料結構規劃）
第3週	基本功能實作（新增/刪除）
第4週	Priority Queue 排序實作
第5週	查詢與篩選功能
第6週	檔案儲存功能
第7週	測試與除錯
第8週	完成報告與展示
五、與課程的關聯
本專題與資料結構課程密切相關，主要應用如下：
Heap（Priority Queue）
用於實現任務優先排序，是本專題核心
Linked List
用於動態儲存任務資料
時間複雜度分析
比較不同資料結構在插入、刪除、查詢的效率
實務應用
將抽象的資料結構應用於真實問題（時間管理）
